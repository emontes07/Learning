Lab 0 — Quick Start: What is Edit with Copilot?
⏱️ 5 minutes 🎯 Setup & orientation 🛠️ No exercise — just read
📂 Files you'll need for this training:
  • Excel-Copilot-Labs-1-to-3.xlsx — used in Labs 1, 2, and 3
  • Excel-Copilot-Labs-4-and-5.xlsx — used in Labs 4 and 5
Download both files and save them to your OneDrive or SharePoint before starting (Copilot does not work with locally-saved files).

Edit with Copilot is an AI assistant built into Excel. Instead of writing formulas or building charts yourself, you describe what you want in plain English — and Copilot makes the change directly on the sheet. You stay in control: every change can be reviewed, kept, or undone.

Key idea: Copilot doesn't just tell you what to do — it does it for you.

Before you start — check these

  • You have a Microsoft 365 Copilot license
  • You're using Excel for the web, Windows, Mac, or iPad (Microsoft 365 version)
  • Your file is saved to OneDrive or SharePoint (not just on your local drive)
  • You have internet connectivity
  • Your data is formatted as an Excel table (or a clean range with headers)
💡 TIP: Press Ctrl+T to convert a range into a table. Copilot performs much better with tables — it understands column names and boundaries automatically.

How to open Copilot in Excel

  1. Open your workbook in Excel.
  2. Click the Copilot icon on the Home tab of the ribbon.
  3. The Copilot pane opens on the right.
  4. Type a natural-language prompt. Copilot processes it and makes changes on the sheet.
  5. If you don't like the result, press Ctrl+Z to undo.
📝 NOTE: Don't see the Copilot icon? Your tenant or license may not have Copilot enabled — check with your IT admin.
🔴 IMPORTANT: Lab 4 uses the new COPILOT() function, which is currently in Preview. You'll need to be on the right Microsoft 365 channel (Beta / Current Channel Preview) for that lab to work. Labs 0–3 and 5 work in standard channels.
Lab 1 — Trust Copilot (it helps you learn)
⏱️ 10 minutes 🎯 Goal: Build confidence by asking Copilot to explain formulas 📄 Sheet: AskQuestions
📂 File for this lab: Excel-Copilot-Labs-1-to-3.xlsx — open this workbook in Excel (saved to OneDrive or SharePoint).

Before Copilot makes changes for you, let it teach you. In this lab you'll ask Copilot questions about Excel formulas — what they do, why they break, and how to write new ones. This is the lowest-risk way to start: Copilot writes nothing on your sheet, it just answers.

Open the workbook and go to the AskQuestions sheet

Exercise 1.1 — Ask "how do I…?"

Open the Copilot pane and try:

💬 Try this prompt
How would I write a formula to split a column with comma separated values into separate columns?

Copilot should explain TEXTSPLIT() and give you a working example.

Exercise 1.2 — Ask "why is this broken?"

Look at cell B28 in the AskQuestions sheet — it shows a #VALUE! error. Ask Copilot why:

💬 Try this prompt
Why is there a #VALUE! Error in B28 with this formula? =TEXTSPLIT(A28,",")

Exercise 1.3 — Ask "what does this formula do?"

💬 Try this prompt
What does this formula do? =IF(ISNUMBER(SEARCH(",",A28)), TEXTSPLIT(A28,","),"")

Exercise 1.4 — Concept questions

Copilot can also teach concepts, not just formulas:

💬 Try this prompt
How can I calculate standard deviation in Excel?
💬 Try this prompt
Remind me what standard deviation means exactly?
💡 TIP: Use Copilot as a tutor when you're stuck. Asking "what does this formula do?" is faster than searching the web — and Copilot knows your data.

Lab 4 — Copilot becomes part of Excel formulas (Preview)
⏱️ 20 minutes 🎯 Goal: Use the new COPILOT() function inside cells 🧪 Preview feature 📄 Workbook: Excel-Copilot-Labs-4-and-5.xlsx
📂 File for this lab: Excel-Copilot-Labs-4-and-5.xlsx — open this workbook in Excel (saved to OneDrive or SharePoint). Lab 4 uses the Feedback, Category, Sentiment, Pivot Table, Transactions, and Customer Feedback sheets.
🔴 IMPORTANT: COPILOT() is currently in Preview. You need a Microsoft 365 Copilot license and the right update channel (Beta / Current Channel Preview). If you get #NAME?, the function isn't available in your Excel build yet.

This is where it gets interesting. Until now, you've used Copilot through a side pane. The new COPILOT() function turns AI into a native Excel formula — chainable with PIVOTBY, TEXTJOIN, FORMULATEXT, spill ranges, and everything else you already know.

Exercise 4.1 — Generate sample data from a prompt

Open the Feedback sheet of Copilot Excel - Command v02. In an empty cell:

💬 Type this formula
=COPILOT("give me 20 reviews for a coffee machine feedback, give me a mix of positive, neutral and negative")

Exercise 4.2 — Summarize a range

💬 Type this formula
=COPILOT("summarize this feedback into a parragraph",B3:B28)

Exercise 4.3 — Categorize against your own category list

Switch to the Category sheet. The categories you can use are in B4:B7.

💬 Type this formula
=COPILOT("Categorize this feedback",C4:C28,"into one of these categories",B4:B7)

Exercise 4.4 — Multi-column output (sentiment + emoji)

Switch to the Sentiment sheet. Headers in D3:F3 describe what columns you want back.

💬 Type this formula
=COPILOT("Categorize this feedback",C4:C28,"into one of these categories",B4:B7,"return these columns",D3:F3)

Exercise 4.5 — Pivot, then ask Copilot to explain its own formula

Switch to the Pivot Table sheet. First build a pivot with the new PIVOTBY function:

💬 Type this formula
=PIVOTBY(D4:D28,F4:F28,F4:F28,COUNTA)

Now have Copilot explain what you just wrote:

💬 Type this formula
=COPILOT("Explain in plain English what this formula is doing",FORMULATEXT(G4))

Exercise 4.6 — Auto-categorize transactions

Switch to the Transactions sheet. Step 1: ask Copilot to propose categories from the data:

💬 Type this formula
=COPILOT("analyze the transactions",B5#,"and provide with a short list of categories I can use to group the expenses")

Step 2: use those categories to label every transaction:

💬 Type this formula
=COPILOT("using this categories",G5#," please assign to each transaction",B5#)

Exercise 4.7 — Regex + urgency rating from messy text

Switch to the Customer Feedback sheet. Generate sample messages, then ask Copilot to extract name + phone + urgency.

💬 Type this formula
=COPILOT("Give me 20 messages for our contact center about feedback related to the coffee machine, each include a phone number and a name")
💬 Type this formula
=COPILOT("Extract the name",A4#)
💬 Type this formula
=COPILOT("return a regex pattern to match",C3,"from this example",A4)
💬 Type this formula
=COPILOT("rate these messages based on how urgent they are as !, !!, !!!, !!!!",A4#)
💡 TIP: The # after a cell reference (e.g. B5#) is the spill operator — it tells Excel "use the entire dynamic array starting here." It's how you pass whole result sets between COPILOT() calls.
Lab 5 — Copilot solves a real business problem
⏱️ 15 minutes 🎯 Goal: Run a full Budget-vs-Actuals variance analysis end to end 📄 Sheet: Financial Report 💼 Finance use case
📂 File for this lab: Excel-Copilot-Labs-4-and-5.xlsx — same workbook as Lab 4, switch to the Financial Report sheet.

You've learned the techniques. Now put them together on a realistic finance scenario: a monthly Profit & Loss statement with budget vs. actuals, variance dollars, and variance percentages. The goal is to produce commentary leadership can actually read.

The data

Open the Financial Report sheet in Copilot Excel - Command v02. It contains:

  • Revenue: Consultancy Income, Domestic Sales, Export Sales, Other Revenue
  • Cost of Goods Sold: COGS, Delivery Costs
  • Gross Profit
  • Expenses: Accounting, Depreciation, Merchant Fees, Motor Vehicle, Postage, Staff, Stationery, Telephony, Training, Travel
  • Net Profit

Every line has Actual, Budget, Variance ($), and Variance %.

Exercise 5.1 — A single summary paragraph

Drop this into an empty cell:

💬 Type this formula
=COPILOT("Analyze the profit and loss statement and give me a summary of notable points",B2:F29)

Exercise 5.2 — Structured commentary across all four lenses

Now ask Copilot for a multi-section analysis — and use TEXTJOIN with double line breaks to format it cleanly:

💬 Type this formula
=TEXTJOIN(CHAR(10)&CHAR(10),TRUE,COPILOT("Analyze the profit and loss statement and give me a summary of notable points organized into these sections: Revenue Analysis, Cost Analysis, Profitability, Expenses, Net Profit. Each section should be one short paragraph.",B2:F29))

Exercise 5.3 — Try a few business questions

Use the Copilot side pane (not the formula) for these — they're follow-up questions on the same data:

💬 Try this prompt
Highlight all rows where the absolute variance percentage exceeds 10% in red
💬 Try this prompt
Create a bar chart showing the top 5 expense categories by absolute variance
💬 Try this prompt
Which line items are driving the Net Profit shortfall the most? List them in priority order.

Stretch: write your own prompt

Imagine you're presenting this to your CFO tomorrow. What's the one prompt you'd ask to get a board-ready takeaway? Write it, run it, and compare with the person next to you.

⚠️ WARNING: Never use Copilot output as the sole basis for a financial decision without independent verification. AI-assisted analysis is a productivity tool — not a substitute for professional judgment. Always sanity-check the numbers it quotes.
💡 TIP: The most productive Copilot users treat it as a conversation. Start broad, refine based on results, and iterate until the output matches what you'd present to your leadership.
Lab 6 — Cowork: one prompt, four deliverables
⏱️ 20–25 minutes 🎯 Goal: Run a multi-artifact Cowork scenario end-to-end 👤 Persona: VP Finance / FP&A Director 💼 Finance use case 🤝 Cowork
📂 File for this lab: EDG_Finance_Performance.xlsx — download this workbook and save it to your OneDrive or SharePoint before starting. It contains Edgewell Personal Care's Q2 FY26 Budget vs Actual data by department × month, key drivers, and an open close-task list.

In Labs 1–5 you used Copilot as a single-step helper inside Excel. Cowork is different — you give Copilot one rich prompt and it produces multiple connected deliverables (a variance workbook, a CFO brief, a QBR deck, and a one-page HTML dashboard) in a single agentic run. The persona below is a VP of Finance closing Q2 FY26 books for Edgewell Personal Care and preparing the variance review for the CFO.

The scenario

Company: Edgewell Personal Care (NYSE: EPC) — ~$2.2B in revenue across three segments (Wet Shave, Sun & Skin Care, Feminine Care) plus DTC brands (Billie, Jack Black). Fiscal year ends September 30.

What's pressuring the quarter:

  • Wet Shave — volume softness in North America private label
  • Sun & Skin Care — residual Banana Boat recall echo holding back NA shelf placement
  • Cuautitlan plant — operator attrition driving unfavorable labor variance
  • A&P spend — running 80 bps over plan to defend share
  • Billie CAC — $28.40 vs $24.00 budgeted
  • Productivity savings — ~380 bps GM tailwind partially offsetting ~195 bps materials inflation

Prompt Title: Q2 FY26 Variance Read & Close-Risk Brief for Edgewell

Exercise 6.1 — The Cowork prompt

Open Copilot Chat (Microsoft 365 Copilot, not just the Excel side pane). Attach EDG_Finance_Performance.xlsx and paste this prompt:

💬 Cowork prompt — paste in full
You are my FP&A partner for Edgewell Personal Care. We are closing Q2 FY26 and I need to walk the CFO through the variance story and the open close risks tomorrow morning.

STEP 1 — ANALYZE
Read EDG_Finance_Performance.xlsx and build a Q2 FY26 variance read:
• Compare Actual vs Budget by department and by month for Q2 FY26 (Jan–Mar 2026 calendar / fiscal Q2).
• Identify the top 5 favorable and top 5 unfavorable variances in absolute dollars AND as a % of plan.
• Call out anything that crosses ±10% vs plan or ±$500K in absolute terms.
• Map variances back to the named drivers in the file (Wet Shave NA volume, Banana Boat recall echo, Cuautitlan attrition, A&P over plan, Billie CAC, productivity savings).
• Flag the three highest-risk items on the open close-task list.

STEP 2 — OUTPUTS (produce all four)
1. EDG_Finance_VarianceDashboard.xlsx — a clean variance workbook with: a summary tab (Actual / Budget / Variance $ / Variance % by department), a drivers tab tying the largest variances to root causes, and a close-risk tab listing the three highest-risk open tasks with owner, ETA, and impact.
2. EDG_Finance_CFO_Brief.docx — a 2-page memo for the CFO: TL;DR up top, Revenue read, Gross Margin read, Operating Expense read, Close-Risk section, and a Recommendation paragraph.
3. EDG_Finance_QBR_Deck.pptx — a 6–8 slide deck for the quarterly business review: title, Q2 headline, segment view (Wet Shave / Sun & Skin Care / Feminine Care / DTC), GM bridge (materials inflation vs productivity), A&P and CAC slide, close-risk slide, ask/recommendation.
4. EDG_Finance_Dashboard.html — a single-page HTML KPI summary the team can open in a browser: top metrics tiles, the segment variance chart, and the three close-risk items.

STEP 3 — TALK TRACK
At the end, give me a 60–120 second executive talk track I can use to open the CFO meeting. Lead with YTD top line, then the Sun & Skin Care NA swing (12–18% below plan), then the GM story (195 bps materials inflation offset by ~380 bps productivity), then the A&P 80 bps over plan, then the three highest-risk close tasks. Keep it crisp.

Rules:
• Only use numbers and drivers that are in the attached file. If something isn't in the data, mark it [ASSUMPTION] in the deliverable.
• Don't invent customer names, contract values, or forecast numbers.
• Where you make a judgment call, say so in one line and explain why.

What Copilot produces

One Cowork run should hand you back four linked files:

  • EDG_Finance_VarianceDashboard.xlsx — summary, drivers, and close-risk tabs
  • EDG_Finance_CFO_Brief.docx — 2-page memo for the CFO
  • EDG_Finance_QBR_Deck.pptx — 6–8 slide quarterly business review deck
  • EDG_Finance_Dashboard.html — one-page browser-friendly KPI summary
🎤 Executive talk track (what "good" sounds like): "YTD we're tracking roughly in line on top line, but Q2 has a Sun & Skin Care NA swing 12–18% below plan — that's the Banana Boat recall echo still holding back shelf placement. Gross margin is the bright spot under the surface: ~195 bps of materials inflation, fully offset by ~380 bps of productivity from Cuautitlan and Sri Lanka. The watch-out is A&P running 80 bps over plan as we defend share, plus Billie CAC at $28.40 vs the $24 we budgeted. On close, three items are live: Cuautitlan labor variance reconciliation, the Banana Boat returns reserve true-up, and the DTC subscription deferred revenue split. I'd like 48 more hours on the first two before we lock the books."
🔴 IMPORTANT — Assumptions vs Facts: Anything Copilot produces that is NOT in EDG_Finance_Performance.xlsx should be marked [ASSUMPTION] in the deliverables. Customer names, future-period forecast figures, and any contract dollars not in the file are out of bounds. Sanity-check every number against the source workbook before sharing with the CFO.

Stretch — optional follow-ups

Once the main run lands, try these as separate Copilot prompts in the same chat:

💬 Stretch prompt 1 — Q3 forecast scenario
Using the Q2 actuals and the named drivers, build a Q3 FY26 forecast scenario with three cases (low / base / high). Assume Sun & Skin Care NA recovers 30% / 50% / 70% of the gap, Cuautitlan attrition stabilizes by mid-Q3, and A&P holds at the current run rate. Show segment revenue, GM%, and Adjusted EBITDA for each case.
💬 Stretch prompt 2 — one-page close-risk memo
Write a one-page close-risk memo to the Controller covering only the three highest-risk open close tasks. For each: what the risk is, the dollar exposure range, the owner, and the recommended path to resolution before we lock the books.
⚠️ WARNING: Cowork output is a starting draft, not a final deliverable. Always reconcile the numbers in the generated workbook against your source ERP data, review the CFO brief for tone and accuracy, and verify chart figures on the deck slides before presenting. The agent is a force multiplier — not the signer-off.

Best practices to take with you

  • Format data as a table before working with Copilot (Ctrl+T).
  • Be specific — column names, thresholds, desired output format.
  • Iterate — start simple, verify, then build complexity.
  • Always review what Copilot wrote before building on it.
  • Keep a prompt journal for recurring tasks.
  • Use Ctrl+Z the moment something looks off.
